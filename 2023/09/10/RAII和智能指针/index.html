Resource Acquisition Is Initialization

note
C++
发布日期:   2023-09-10
文章字数:   336

资源管理

本质特征：

利用对象的声明周期管理资源

特点：1、在对象创建时托管资源
2、在对象销毁时，释放资源
3、提供若干访问资源的额方法
4、一般不能进行复制或者赋值，表现的是对象语义

对象语义：表示在世界中客观存在，例如批量生产的计算机或者汽车，具体针对某一个计算机或者汽车表现的是对象语义，这个汽车是独一无二存在，不可复制

值语义：可以进行复制或者赋值

代码

#include 

using std::endl;
using std::cout;

template
class RAII 
{
public:
    //2、定义类型的构造函数和析构函数
    RAII(Type *p)
    :_p(p)
    {
        cout << "RAII(*p)" << endl;
    }
    
    ~RAII(){
        if(_p){
            delete _p;
        }
        cout << "~RAII()" << endl;
    }
    
    //3、运算符重载
    Type * operator->(){
        return _p;
    }
    
    Type & operator*(){
        return *_p;
    }
    
    Type* get() const 
    { return _p; }
    
    //4、为了体现对象语义，不能复制或者赋值，即将拷贝构造和赋值运算符私有化或者删除
    RAII(const RAII &) = delete;
    RAII & operator=(const RAII& rhs) = delete;
    
private:
    //1、定义一个模板类型指针
    Type * _p;    
};

//自定义类型
class Point{
public: 
    Point(int x, int y)
    :_x(x)
    ,_y(y)
    {
        cout << "Point(x, y)" << endl;
    }
    
    ~Point()
    {
        cout << "~Point()" << endl;
    }
    
private:
    int _x;
    int _y;

};

int main()
{
    RAII raii(new Point(3, 4)); //用RAII托管类型
    
}

四种智能指针

1、auto_ptr （C++17中已被弃用）

  • 源码位置：/usr/include/c++/7/memory ==> /usr/include/c++/7/backward/auto_ptr.h
  • 弃用原因：
    • 在auto_ptr源码中，首先没有禁用拷贝构造和赋值运算符，并且在源码中当一个
    • auto_ptr p1 = p2 的时候，源码中p2已经被置为nullptr,不能够再进行使用了
    • 导致错误

2、unique_ptr

unique_ptr禁用了拷贝构造和赋值，但是保留了移动构造函数。

unique_ptr getValue()
{
  unique_ptr up2(new Point(12, 13));
  return up2
}

unique_ptr up2 = getValue(); //此时调用的是移动语义

//将up2存储到容器中
vector> numbers;
numbers.push_back(up2) //error 这个时候会调用拷贝构造
numbers.push_back(std::move(up2)) //ok。但此时up2为nullptr

up2.reset(new Point(13, 14)) //重新托管一个值

3、share_ptr

  • 1、针对于unique_ptr的所有属性share_ptr都适配
  • 2、share_ptr既有拷贝构造有又有赋值运算符。（都有移动语义）
  • 3、当将sp使用移动语义进行赋值之后，其本身不再托管原来的对象。
shared_ptr sp1(new Point(1, 1));

vector> points;
points.push(sp); //此时会调用拷贝构造函数，sp1的引用计数+1
points.push(std::move(sp)); // 此时会调用移动构造函数，sp的引用计数，sp1不再托管原来的对象
  • 4、对于shared_ptr当已有use_count为1时，再进行reset操作，先将use_count减去1，如果为0则释放资源，否则不进行任何操作。
  • 5、shared_ptr的问题：循环引用
    • 例如：A类中有一个shared_ptr指向B类，B类中有一个shared_ptr指向A类, use_count无法归0，这样就会导致内存泄漏。
    • 解决方法：使用weak_ptr来解决循环引用的问题。

4、weak_ptr

  • 1、弱引用智能指针
  • 2、特点：
    • 1、weak_ptr进行赋值或赋值时，不会引起引用计数增加
    • 2、==他不能访问托管对象的资源==，没有重载解引用和成员访问运算符
    • 3、他知道所托管对象是否还存活，使用expired()方法来判断
    • 4、想访问资源必须通过lock()方法进行提升，来获取shared_ptr对象
    • 5、可以查看托管对象的引用计数
  • 3、循环引用解决方法：
    • 将A 或者 B中的引用修改为weak_ptr

删除器

struct MyFileDeleter{

2、讲解

unique_ptr是在模板中定义了一个删除器，传递的是一个类型，在这个类中实现一个删除器类的实例化，进行删除

unique_ptr up(fopen("demo.txt", "a+"));

在shared_ptr中，则是在构造函数中实现了一个构造函数，需要传递具体的对象进去


shared_ptr sp(fopen("demo.txt", "a+"), MyFileDeleter());

智能指针的错误使用

1、将一个堆空间指针交给两个智能指针管理

Point * pt = new Point(1, 2);

unique_ptr up1(pt);
unique_ptr up2(pt); //错误

2、将一个已经被智能指针托管的内容使用reset交给其他智能指针托管


shared_ptr sp1(new Point(1, 2));
shared_ptr sp2(new Point(12, 23));

sp2.reset(sp1.get()); //这行代码会引发问题。
// sp1.get() 返回的是 sp1 所管理对象的原始指针，
// 当调用 sp2.reset(sp1.get()) 时，sp2 会接管该原始指针，
// 这就导致两个 shared_ptr（sp1 和 sp2）管理同一个对象，然而它们并不知道彼此的存在。

3、对象中返回一个智能指针

希望在类内部获取本对象的智能指针
1、先继承自辅助类 enable_shared_from_this
2、调用成员函数shared_from_this()

class Point
: public std::enable_shared_from_this
{
public:
  Point(int x, int y)
  :_x(x)
  ,_y(y)
  {
  }
  
  ~Point()
  {
  
  }
   // Point * addPoint(Point &pt) //error返回的是原生指针，无法再托管给其他智能指针
  shared_ptr addPoint(Point &pt){
      _x += pt._x;
      _y += pt._y;
      // return shared_ptr(this) //error 犯了第二种错误，this是原生指针
      return shared_from_this()
  }

private:
    int _x;
    int _y;

};
